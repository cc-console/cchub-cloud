从安装到手机访问Get started

把桌面上的 Claude / Codex / Gemini 会话带到手机、平板、任意浏览器,实时查看、随手接管。Bring your desktop Claude / Codex / Gemini sessions to your phone, tablet, or any browser — watch live and take over anytime.

1

安装 cc-consoleInstall cc-console

桌面 app(推荐):从 cc.cchub.cloud 下载对应平台安装包,双击安装。Desktop app (recommended): download the installer for your platform from cc.cchub.cloud and run it.

命令行 / 服务器(macOS / Linux):Command line / server (macOS / Linux):

curl -fsSL https://github.com/cc-console/releases/releases/latest/download/install.sh | sh
macOS 首次打开提示「已损坏」? 这是未签名 app 被 Gatekeeper 拦截。把 app 拖到「应用程序」后执行一次:“cc-console is damaged” on first open (macOS)? The unsigned app is blocked by Gatekeeper. Drag it into Applications, then run once:
xattr -dr com.apple.quarantine /Applications/cc-console.app

运行依赖:macOS/Linux 需要 tmux(brew install tmux / sudo apt install tmux)和你要用的 CLI(claude / codex / gemini)在 PATH 中;Windows 自带会话引擎(无需 tmux)。开远程隧道还需 cloudflared(mac brew install cloudflared,linux 见下方命令),桌面 app 也不自带;纯本机使用则不需要。Requirements: macOS/Linux need tmux (brew install tmux / sudo apt install tmux) and a CLI (claude / codex / gemini) on PATH; Windows ships its own session engine (no tmux). Remote tunnels also need cloudflared (mac: brew install cloudflared; Linux: see below) — not bundled with the desktop app; local-only use doesn't need it.

一键连接远程Connect remote

回到 app,点顶部 🌐 远程 Remote → 连接。浏览器会打开授权页,点「批准」即可 —— app 自动拿到设备码、配置隧道,无需任何复制粘贴。Back in the app, click 🌐 Remote → Connect. A browser approval page opens — click Approve, and the app fetches the device code and sets up the tunnel automatically, no copy-pasting.

打开远程页Open Remote

连接成功后,你会得到一个稳定地址,例如 https://你的名字.cchub.cloud。Once connected you get a stable address like https://your-name.cchub.cloud.

在 Ubuntu 服务器上? 不走上面的「连接」按钮,改用设备码绑定 —— 先在服务器配好代理让它能连上 AI(参考 clash-for-linux-install),在 账号页点「生成设备码」拿到 ccd_…,然后在服务器执行:On an Ubuntu server? Skip the Connect button and bind with a device code — first set up a proxy so the box can reach the AI (see clash-for-linux-install), click “Generate device token” on the account page for ccd_…, then on the server run: 
cc-console link          # 粘入 ccd_… 设备码# paste the ccd_… device code
cc-console               # 启动，managed 隧道自动拉起# start; the managed tunnel comes up
浏览器打开日志里的 https://你的名字.cchub.cloud/?token=… 即可。设备码会保存,下次开机自动重连。Open the https://your-name.cchub.cloud/?token=… printed in the logs. The code is saved and reconnects on the next boot.

常驻(断开 SSH 也不掉,推荐用 systemd 用户服务):Keep it running after you disconnect (recommended — a systemd user service):

mkdir -p ~/.config/systemd/user
cat > ~/.config/systemd/user/cc-console.service <<EOF
[Unit]
After=network-online.target
[Service]
ExecStart=%h/.local/bin/cc-console
Restart=on-failure
[Install]
WantedBy=default.target
EOF
systemctl --user enable --now cc-console
sudo loginctl enable-linger $USER

在手机 / 平板上访问Open on your phone

在手机浏览器打开你的稳定地址(首次带上日志里给出的 ?token=…,之后会记在 cookie 里)。你会看到和桌面完全一致的实时终端 —— 同一个会话,随手接管。Open your stable address in a phone browser (first time, include the ?token=… from the log; after that it's remembered in a cookie). You'll see the exact same live terminal as the desktop — the same session, ready to take over.

  • 多窗口并排:Claude / Codex / Gemini 一目了然,按状态颜色区分。Windows side by side: Claude / Codex / Gemini at a glance, colour-coded by status.
  • 断线自动重连,tmux 会话不丢上下文。Auto-reconnect on drop; the tmux session keeps its context.

  • 通知:后台会话停下等你决定时,会用声音 + 浏览器通知 + 标签角标提醒你。Alerts: when a background session stops for your decision, it pings you with a sound, a browser notification, and a tab badge.

常见问题FAQ

「已损坏,无法打开」怎么办?“cc-console is damaged” — how do I open it?

未签名 app 被 Gatekeeper 拦。执行 xattr -dr com.apple.quarantine /Applications/cc-console.app 后即可打开。命令行安装(install.sh)不会有这个问题。The unsigned app is blocked by Gatekeeper. Run xattr -dr com.apple.quarantine /Applications/cc-console.app once, then open it. The CLI install (install.sh) avoids this entirely.

提示需要订阅 / 无法生成设备码?It says a subscription is required?

远程访问是付费功能。去 订阅页 开通后重试。Remote access is a paid feature. Activate it on the subscribe page, then retry.

连接成功但打不开地址?Connected, but the address won't open?

先确认在账号页设置了 handle;地址形如 你的名字.cchub.cloud。新地址的 DNS / 证书通常几秒内生效。First make sure you set a handle on the account page; the address looks like your-name.cchub.cloud. DNS / certs for a new address usually take a few seconds.

不想要账号,只想临时分享一下?No account — just want to share temporarily?

远程页 → 高级 → 「临时链接」,生成一次性 *.trycloudflare.com 地址(每次重启都变,无需账号)。Remote page → Advanced → Quick link generates a throwaway *.trycloudflare.com address (changes on every restart, no account needed).
